2. Struktur Skema Database Firestore & Aturan Keselamatan

Skema Ternormalisasi & Aturan RABC (Role Based Access Control) Gred Institusi Kewangan.

Firebase Security Rules (firestore.rules)

rules_version = '2'; service cloud.firestore { match /databases/{database}/documents font-semibold { // Helper Functions function isAuthenticated() { return request.auth != null; } function getUserData() { return get(/databases/$(database)/documents/users/$(request.auth.uid)).data; } function isSuperAdmin() { return isAuthenticated() && getUserData().role == 'super_admin'; } function isEmployer(companyId) { return isAuthenticated() && getUserData().role == 'employer' && getUserData().companyId == companyId; } function isEmployee(userId) { return isAuthenticated() && request.auth.uid == userId; } // Collection Rules match /companies/{companyId} { allow read, write: if isSuperAdmin(); allow read: if isAuthenticated() && getUserData().companyId == companyId; } match /users/{userId} { allow read, write: if isSuperAdmin(); allow read, update: if isEmployee(userId); allow read, create, update: if isAuthenticated() && getUserData().role == 'employer' && getUserData().companyId == request.resource.data.companyId; } match /attendance/{attendanceId} { allow read, write: if isSuperAdmin(); allow create, read: if isAuthenticated() && request.resource.data.userId == request.auth.uid; allow read, update: if isAuthenticated() && isEmployer(resource.data.companyId); } match /leave/{leaveId} { allow create, read, update: if isAuthenticated() && (request.resource.data.userId == request.auth.uid || resource.data.userId == request.auth.uid);
